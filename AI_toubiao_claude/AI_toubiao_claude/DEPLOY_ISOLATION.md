# 部署隔离说明

## 🔒 隔离措施

为确保本次部署不影响服务器上已有的其他系统，我们采用了以下隔离措施：

### 1. 端口隔离
- **使用端口**: `8080`（而非默认的80端口）
- **原因**: 避免与现有系统占用80端口冲突
- **访问地址**: `http://服务器IP:8080`

### 2. 容器隔离
- **容器名称**: `ai-toubiao-web`（唯一命名）
- **镜像名称**: `ai-toubiao-demo:latest`
- **不会影响**: 其他Docker容器的运行

### 3. 网络隔离
- **网络名称**: `ai-toubiao-network`（独立网络）
- **网络模式**: bridge（独立桥接网络）
- **不会影响**: 其他容器的网络配置

### 4. 目录隔离
- **部署目录**: `/opt/ai-toubiao-demo`
- **不会影响**: 其他系统的文件目录

### 5. 资源隔离
- **CPU/内存**: Docker自动管理，不影响其他容器
- **磁盘**: 仅使用项目所需空间

## ✅ 部署前检查

部署脚本会自动检查：
1. ✅ 端口8080是否可用
2. ✅ 现有容器列表（仅查看，不修改）
3. ✅ Docker环境（如果未安装会自动安装，不影响现有容器）

## 🚫 不会影响的内容

- ✅ 现有Docker容器（不会停止、删除或修改）
- ✅ 现有服务端口（80端口不会被占用）
- ✅ 现有网络配置
- ✅ 现有文件系统（仅使用独立目录）
- ✅ 系统服务（不会修改系统配置）

## 📝 部署后的验证

部署完成后，可以通过以下命令验证隔离性：

```bash
# 查看所有容器（确认现有容器仍在运行）
docker ps -a

# 查看端口占用（确认8080端口被使用，80端口未被占用）
netstat -tlnp | grep -E ':(80|8080)'

# 查看网络（确认独立网络已创建）
docker network ls

# 查看本项目的容器状态
cd /opt/ai-toubiao-demo
docker-compose ps
```

## 🔧 如需修改端口

如果8080端口也被占用，可以修改 `docker-compose.yml`：

```yaml
ports:
  - "8081:80"  # 改为其他可用端口
```

然后重新部署即可。

## 📞 问题排查

如果部署后发现问题：

1. **检查端口冲突**
   ```bash
   netstat -tlnp | grep 8080
   ```

2. **检查容器状态**
   ```bash
   docker ps -a | grep ai-toubiao
   ```

3. **查看部署日志**
   ```bash
   cd /opt/ai-toubiao-demo
   docker-compose logs
   ```

4. **停止本项目（不影响其他系统）**
   ```bash
   cd /opt/ai-toubiao-demo
   docker-compose down
   ```

---

**总结**: 本次部署完全独立，不会影响服务器上的任何现有系统！
